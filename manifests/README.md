# Manifests

This directory contains GDC manifests used to download whole-slide image data. Each manifest is logged below with the filters used
to generate it, the resulting size, and why that particular subset was chosen.

---

## gdc_manifest_full_luad_dx.txt

**Date generated:** 2026-06-23  
**Source:** [GDC Data Portal Repository](https://portal.gdc.cancer.gov/analysis_page?app=Downloads)  
**Result:** 478 cases, 541 files  

| Filter                 | Value            |
|------------------------|------------------|
| Program                | TCGA             |
| Project                | TCGA-LUAD        |
| Access                 | Open             |
| Data Format            | svs              |
| Data Type              | Slide Image      |
| Experimental Strategy  | Diagnostic Slide |

**Why:** Full open-access LUAD diagnostic (FFPE) slide cohort. Restricted to
Diagnostic Slide to exclude frozen Tissue Slide sections (`TS`/`BS` barcodes),
which carry ice-crystal artifacts and a domain gap from the FFPE data UNI/CONCH
were trained on. This is the base manifest all downstream subsets are sampled
from.

**Generated by:** manual filter selection in GDC Repository UI, no script.

## gdc_manifest_pilot100_dx.txt

**Date generated:** 2026-06-24  
**Source:** sampled from `gdc_manifest_full_luad_dx.txt`  
**Result:** 100 cases, 100 files

| Filter                 | Value                                  |
|------------------------|----------------------------------------|
| Base manifest          | gdc_manifest_full_luad_dx.txt          |
| Cases                  | only those in matched_clinical_slides  |
| Slides per case        | 1 (first diagnostic slide, DX1)        |
| Sample size            | 100                                    |
| Stratified by          | event (death vs. censored)             |
| Seed                   | 42                                     |
| Event balance          | 65 censored / 35 deaths                |

**Why:** Small pilot for end-to-end pipeline testing. Restricted to
cases present in `matched_clinical_slides.csv` so every slide has a usable
survival outcome (`event`, `time`). One diagnostic slide per case keeps
images == patients == 100, avoiding correlated same-outcome duplicates that
would leak signal. Stratified by `event` so the pilot mirrors the cohort's
~35% death rate rather than landing all-censored or all-deaths, which would
make survival metrics (Cox partial likelihood, concordance) meaningless.
